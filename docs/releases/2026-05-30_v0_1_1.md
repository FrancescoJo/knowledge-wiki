## 0.1.1 (2026-05-30)

> First functional release. Implements the complete authentication and user management foundation, an HTMX-driven frontend login flow, and the project's core backend architecture conventions.

### Upgrade Steps

* No upgrade steps required — this is the initial release.

### Breaking Changes

* None.

### New Features

* **auth:** JWT-based login, logout, and token refresh via Spring Security filter chain
* **auth:** Login credential management with form-encoded request support
* **user:** User registration endpoint and user CRUD API
* **frontend:** `omnimemo` TypeScript module with login popup UI — password toggle, HTMX + JSON auth wire-up
* **frontend:** English/Korean i18n with browser-aware locale resolution and flag-icon dropdown
* **frontend:** Authenticated user email displayed in header with logout dropdown
* **api:** OpenAPI schema documentation available for local/alpha builds
* **infra:** Docker Compose for local PostgreSQL and server startup quickstart helper

### Bug Fixes

* None.

### Improvements

* **backend:** Response envelope and structured error codes introduced across all API endpoints
* **backend:** User display info cached per-session to avoid per-request DB lookups
* **backend:** `UseCase` pattern replaces `Service`; domain-first package structure adopted
* **backend:** `core.util` parsing utilities and controller interface/impl pattern with API v1 versioning
* **backend:** `AesGcmCipher` and `HmacBlindIndex` utilities for sensitive field storage
* **build:** Kotlin upgraded to 2.0.21, Detekt to 1.23.8
* **build:** Build phases introduced with binary-level API doc isolation
* **test:** Testcontainers instance shared across repository medium tests; WebMvcTest medium tests added for Auth and User controllers; assertions migrated to Kotest
